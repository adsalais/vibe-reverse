Audience: an expert reverse engineer. Put the most important things first.

Executive summary

  • Outcome / verdict: solved — the key for any username is recoverable in O(n).
  • What it is: ELF 64-bit PIE x86-64 CTF crackme; a <user> <key> check, not packed.
  • Top findings (3–5):
  • confirmed The check is want[i] = username[i] + 1 then strcmp(want, key) — a pure, invertible byte transform.
  • confirmed No packing/obfuscation (entropy 1.79); standard hardening (PIE/NX/RELRO/canary).
  • confirmed Keygen key[i] = username[i] + 1 is verified against the binary.
  • Headline IOCs: none (benign in-house fixture).

Key findings

main reads argv[1] (user) and argv[2] (key). It computes a want buffer where each byte is the corresponding username byte plus one, then accepts iff strcmp(want, key) == 0. There is no hashing, encryption, or hidden state.

  • Solve (re-scripting): a 1-line pure keygen, unit-tested ("AB"→"BC") and verified against the real binary (crackme1 AB BC → Correct!).
